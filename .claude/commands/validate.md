# 코드 검증

현재 작업 중인 파일 또는 지정된 파일의 맑은프레임워크 규칙 위반을 검증한다.

## 대상
$ARGUMENTS 가 있으면 해당 경로의 파일을, 없으면 최근 수정된 JSP/HTML/Java 파일을 검증한다.

## 절차

1. 대상 파일을 읽는다.
2. 파일 확장자에 따라 file_type을 결정한다:
   - `.jsp` → `jsp`
   - `.html` → `html`
   - `.java` (DAO) → `dao`
3. MCP `validate_code(code, file_type)`를 호출하여 검증한다.
4. 위반 사항이 있으면:
   - 각 위반의 ruleId로 `get_rules(rule_id)`를 조회하여 올바른 사용법을 확인한다.
   - 수정 방안을 제시한다.
5. 위반 사항이 없으면 "검증 통과"를 보고한다.

여러 파일이 대상이면 모든 파일을 병렬로 검증한다.
